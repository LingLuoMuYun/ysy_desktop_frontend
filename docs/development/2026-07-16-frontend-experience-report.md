# 2026-07-16 前端经验报告

## 背景

本次前端工作集中在项目页。目标是让项目列表布局稳定、项目详情弹窗可用，并对项目页代码做适度拆分，降低后续接入真实接口和详情子列表的维护成本。

## 修复的问题

### 1. 项目列表高度不正确

现象：

- 页面只有一个项目卡片时，列表区域高度只有内容高度。
- 列表没有填满搜索栏下方到分页栏之间的剩余空间。
- 单卡片可能贴近分页栏底部，不符合列表从顶部排列的预期。

根因：

- `.project-page` 的 `display: flex` 被全局 `.workbench-page` 的 `display: grid` 覆盖。
- 两者特异度相同，最终加载顺序使全局样式胜出。
- 项目页实际仍按 Grid 渲染，第三个子元素进入隐式 Grid 行，隐式行高度按内容撑开。

处理方式：

- 使用 `.workbench-page.project-page` 提高页面级样式特异度。
- 明确项目页为纵向 Flex。
- 列表外壳和滚动区域使用 `flex: 1` 与 `min-height: 0`。
- 卡片列表统一从顶部排列。

经验：

- 布局问题不要只看源码中是否写了样式，要在 DevTools Computed 面板确认最终生效值。
- 页面级覆盖全局基类时，应显式提高特异度，而不是依赖 CSS 导入顺序。
- Flex 子项要在可滚动容器链路上补 `min-height: 0`，否则内容可能撑破或无法正确收缩。

### 2. 项目页组件过大

现象：

- 列表、筛选、分页、创建项目、删除项目、详情弹窗都堆在 `ProjectsPage.tsx`。
- 页面组件同时承担 API 调用、表单状态、弹窗状态和展示结构。
- 后续继续增加详情 Tab 数据时会难以维护。

处理方式：

- 拆出 `useProjectList` 管理列表状态。
- 拆出 `ProjectCreateDialog`、`ProjectDeleteDialog`、`ProjectDetailDialog`。
- 拆出 `projectFormHelpers` 复用表单字段和状态展示。
- `ProjectsPage.tsx` 只保留页面组装、Toast、弹窗显隐和 API 桥接。

经验：

- Page 组件适合做编排，不适合承载完整业务流程。
- 对话框是天然的 feature 组件边界。
- Hook 负责服务端状态和 URL 同步，能减少页面主体的状态噪声。

### 3. 项目详情弹窗过大

现象：

- 初版弹窗宽高偏大，占据过多工作区空间。
- 字体、Tab、按钮尺寸接近截图但不适合当前桌面工作台密度。
- 小窗口下可用空间不足。

处理方式：

- 弹窗最大宽度收敛到 680px。
- 固定大高度改为 `max-height`，按视口自适应。
- 标题从 29px 调整到 24px，字段和值从 21px 调整到 15px。
- 图标从 60px 调整到 48px。
- 底部按钮高度从 48px 调整到 40px。
- 增加 `max-width: 760px` 的响应式规则：标题换行、Tab 横向滚动、按钮换行。

经验：

- 参考图不应机械等比复刻，需要结合当前应用的信息密度和容器尺寸调整。
- 弹窗应优先使用 `max-height` 和内部滚动，而不是固定高度。
- Tab 在窄宽度下横向滚动比强制挤压更稳定。

### 4. 项目数据集数量类型不一致

现象：

- `ProjectSummary.datasetCount` 原为字符串。
- 后端接口返回数字。
- Mock 数据使用带单位的字符串，如 `45,280 条`。

处理方式：

- 将 `datasetCount` 改为数字类型。
- API 映射直接使用 `project.datasetCount ?? 0`。
- Mock 数据改为数字。

经验：

- 展示单位应放在 UI 层，不应混入领域类型。
- 领域类型越贴近接口契约，后续统计、排序和筛选越简单。

### 5. 项目接口代理缺失

现象：

- 前端服务已有项目 API 客户端，但 Vite proxy 未覆盖 `/api/projects`。
- 开发环境中如果浏览器直连业务后端失败，项目接口缺少同源代理回退路径。

处理方式：

- 在 `vite.config.ts` 的 dev 和 preview proxy 中加入 `/api/projects`。
- 项目服务默认后端地址切换到当前业务服务地址。

经验：

- 新增业务 API 客户端时，需要同步检查 Vite dev proxy 和 preview proxy。
- 默认地址、环境变量和代理规则要成组维护。

### 6. 刷新按钮缺少反馈

现象：

- 点击顶部刷新按钮后，用户无法判断是否触发。
- 快速连续点击可能重复触发刷新。

处理方式：

- 增加 `isRefreshing` 状态。
- 点击后按钮禁用 600ms。
- 添加 `icon-spin` 旋转动画。
- 组件卸载时清理定时器。

经验：

- 只要操作不是即时产生可见变化，就需要局部反馈。
- 前端定时器必须在组件卸载时清理。
