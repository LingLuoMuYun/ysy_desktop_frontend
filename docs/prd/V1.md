# V1 PRD

来源：`V1.pdf`

整理日期：2026-05-18

## 产品定位

智算桌面是一个面向本地 AI 实验的桌面应用。它管理用户的代码、数据、环境、训练任务、模型文件、服务和实验结果，并通过 AI 助手把复杂操作转化为可执行任务。

AI 助手不是普通聊天框，而是产品的统一操作中枢。所有核心功能都必须同时提供图形界面入口和 AI 助手入口。

## 核心原则

### 所有功能必须有两种入口

每个功能都要支持两种使用方式。

方式一：用户点击操作。

示例流程：

```text
项目管理 -> 新建项目 -> 选择代码目录 -> 选择数据集 -> 绑定环境/自动创建环境 -> 创建完成
```

方式二：AI 助手操作。

示例输入：

```text
帮我创建一个 YOLOv8 实验项目，代码目录是 D:/code/yolo，数据集目录是 D:/dataset/coco128，使用 Python 3.10 和 PyTorch 环境。
```

或者：

```text
帮我创建一个 yolo8 的实验项目，并帮我起一个训练任务。
```

AI 需要通过交互补齐缺失信息，或者在可控范围内自动补全。

AI 助手应能自动完成：

- 识别用户意图
- 检查目录
- 创建项目
- 推荐环境
- 检查依赖
- 生成训练入口
- 提醒用户缺失项
- 给出下一步操作建议

### AI 助手是操作中枢

AI 助手承担三类角色。

操作助手：

- 创建项目
- 创建环境
- 安装依赖
- 启动训练
- 启动模型服务
- 查看日志
- 停止任务
- 处理数据，并明确能力边界

诊断助手：

- 依赖安装失败
- CUDA 不可用
- 显存不足
- 训练脚本路径错误
- 数据集格式不符合要求
- 端口被占用
- 模型服务启动失败

分析助手：

- 对比多次训练结果
- 总结 loss 曲线变化
- 解释 mAP、precision、recall
- 生成实验报告
- 给出下一轮调参建议

## 核心模块

### 模块一：项目管理

定位：管理用户的代码、数据、环境、任务和实验结果。

核心能力：

- 新建项目
- 导入已有项目
- 扫描本地项目结构
- 绑定代码目录
- 绑定数据目录
- 绑定或新建运行环境
- 管理训练任务
- 管理模型文件
- 管理实验记录
- 对比实验结果
- 生成实验报告

导入已有项目时需要处理混乱目录：

- 有的项目有 `requirements.txt`，有的没有
- 有的是 YOLO 项目，有的是普通 PyTorch 项目
- 有的代码、数据、权重在同一目录
- 有的代码、数据、权重分散在多个目录

### 模块二：环境配置

定位：解决用户最痛苦的环境安装问题。是否让用户感知环境细节，需要在设计中进一步明确。

核心能力：

- 检测本机硬件
- 检测 CUDA、GPU、Python、Docker、WSL 等
- 创建 Python 环境
- 安装 `requirements.txt`
- 无 `requirements.txt` 时扫描 `import`
- AI 辅助识别缺失依赖
- 记录环境快照
- 环境复用
- 环境修复

### 模块三：模型训练

定位：让用户用最少配置完成深度学习或轻量大模型训练。

深度学习训练支持：

- YOLO 系列
- 图像分类
- 目标检测
- 简单自定义训练脚本

核心能力：

- 选择训练脚本
- 选择数据集
- 选择环境
- 配置训练参数
- 启动训练
- 查看日志
- 查看指标
- 保存权重
- 训练失败诊断
- 多实验对比

大模型训练定位：

- 轻量级入口
- 本地小规模微调验证
- 服务器训练任务预留入口

预留支持：

- 本地小模型训练 Demo
- LoRA / QLoRA 入口
- 服务器训练入口
- 云端资源提交入口

### 模块四：模型服务

定位：让训练好的模型可以被启动、测试和调用。

深度学习模型服务能力：

- 选择权重文件
- 选择推理脚本
- 启动本地服务
- 上传图片测试
- 查看推理结果
- 查看服务地址
- 查看 API 文档
- 停止服务

大模型服务第一阶段能力：

- 启动本地小模型服务
- 配置模型路径
- 配置端口
- 查看接口地址
- 简单对话测试
- 后续对接服务器模型服务

AI 操作示例：

```text
帮我把刚刚训练出来的 best.pt 启动成一个检测服务。
```

系统应自动识别：

- 当前项目
- 最新训练结果
- `best.pt` 路径
- 推荐服务配置
- 启动服务
- 返回访问地址

### 模块五：AI 助手

定位：产品的统一操作入口。AI 助手不应该只是右下角聊天框，而应该贯穿所有页面。

全局 AI 助手：

- 在整个桌面应用右侧常驻
- 用户可以随时发起项目、环境、训练、诊断、分析和报告任务

页面内 AI 助手：

- 每个模块都有上下文能力
- 例如用户在环境配置页面时，AI 默认知道当前环境状态
- 用户问“为什么这里显示 CUDA 不可用？”时，AI 应读取当前页面状态并解释

任务执行面板：

AI 执行操作时不能只返回文字，而应该展示任务步骤。

示例：

```text
正在执行：创建 YOLOv8 项目
√ 检查代码目录
√ 检查数据集目录
√ 检测 Python 版本
√ 创建运行环境
× 安装依赖失败：torch 版本与 CUDA 不匹配

建议操作：
1. 安装 CPU 版本 PyTorch
2. 更换 CUDA 版本
3. 使用已有环境
```

## 首页 PRD

首页需要展示：

- 当前设备资源情况
- 最近项目和任务
- AI 对话入口
- 模块快捷入口

## 项目管理 PRD

项目管理应包含：

- 新建项目：快速创建新的实验项目，并初始化基础配置
- 导入项目：将已有代码工程接入平台，生成项目档案
- 项目概览：集中查看项目状态、标签和最近动态
- 代码与数据绑定：统一管理项目的代码目录、数据集和依赖资源
- 训练任务管理：查看、创建和跟踪项目下的训练任务进度
- 模型文件管理：集中管理模型权重文件、导出文件和版本记录
- 实验结果分析：对比训练指标，生成实验报告并进行效果分析
- AI 项目助手：通过 AI 对话完成项目操作、问题诊断和任务管理
