# Design: stack-go-python-ai

## 总体设计

本次只更新技术规划，不初始化工程。新的服务边界如下：

- 前端：React + TypeScript 方向保持不变
- Go 后端：业务 API、本地执行、任务状态、日志采集、环境检测、训练任务和模型服务管理
- Python AI 助手：意图识别、上下文整合、工具编排、诊断、报告生成和模型生态能力
- 数据库：暂未确定，M0 阶段完成选型

## 服务边界

Go 后端是业务状态和本地执行的主要事实源。Python AI 助手不直接维护关键业务状态，不绕过 Go 后端执行高风险系统操作。

Go 与 Python 之间通过明确契约通信，候选方式：

- HTTP
- gRPC
- 本地 RPC

最终方式在工程初始化 spec 中确定。

## 数据库策略

数据库暂未确定，因此在工程初始化前需要补充数据库选型记录。候选方向包括：

- SQLite：适合本地单机桌面应用
- PostgreSQL：适合后续多人协作和复杂查询
- MySQL：适合团队已有运维经验时采用

业务代码应通过 repository / storage 抽象隔离具体数据库。
