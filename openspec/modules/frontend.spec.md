# Frontend Module Spec

本文件定义桌面智算前端与桌面端交互约束。当前仓库前端位于 `frontend/`。

## 技术边界

- 前端使用 React + TypeScript。
- 桌面壳使用 Electron。
- Electron 主进程、预加载脚本、渲染进程和桌面能力封装必须分层清晰。
- 前端不得直接执行训练、环境修改、服务启动等本机高风险动作，必须调用后端受控 API。

## 页面布局

全局区域：

- 左侧导航：固定承载首页、项目、任务、数据、模型、设置。
- 顶部窗口栏：桌面窗口控制和本地软件入口。
- 右侧 AI 助手：项目、任务、数据、模型、设置模块常驻；首页使用居中对话入口。

工作台页面必须优先信息密度、可扫描性和稳定布局，不做营销式落地页。

## 六大模块 UI 规则

首页：

- 展示本机资源概览、最近项目、最近任务和 AI 对话入口。
- 首页不展示 Git 仓库、分支、commit、diff 等管理能力。

项目：

- 支持创建、导入、查看项目详情、绑定数据和环境。
- 项目表单不要求用户填写训练脚本入口；训练入口属于 Task。

任务：

- 同时展示训练任务和推理服务。
- 任务详情必须展示状态、阶段、日志、失败原因和下一步建议。
- 启动、停止、删除、复制为新任务必须有清晰入口和确认规则。

数据：

- 导入本地数据集时只记录路径和元信息，不复制大文件。
- 展示检查摘要、异常项和可由 AI 分析的上下文。

模型：

- 导入模型资产时只记录路径、格式、大小、来源和检查结果。
- 从模型资产创建推理服务后，运行状态进入任务模块。

设置：

- 管理运行环境和 AI 助手模型配置。
- API Key 必须脱敏展示。
- 环境创建、依赖安装和环境修复必须按高风险操作确认。

## 表单交互规则

- 必填字段、系统生成字段、可选字段必须区分展示。
- 所有任务型操作必须有 loading、success、failed、cancelled 或等价状态。
- 失败时必须展示可读错误原因和下一步建议。
- AI 生成的字段建议不得自动提交；用户确认后才写入。

## AI 助手面板规则

- 页面可把当前项目、任务、数据集、模型资产、环境、日志片段或表单字段作为上下文传给 AI。
- 上下文必须轻量展示，支持清除或更换。
- AI 推荐问题应出现在对话消息或助手面板中，不替代真实表单和按钮。
- AI 发起 `action_request` 时，前端必须展示可审计步骤和确认入口。

## 高风险确认

确认弹窗至少包含：

- 操作名称。
- 影响对象。
- 是否启动或停止本机进程。
- 是否修改文件、环境或产品记录。
- 是否可撤销。
- 确认和取消动作。

高风险操作不得只用 toast 表示结果。
