# 修复 AI 表单渲染和对话栏横向溢出

## 背景

AI 生成“学生请假表”等表单时，输出可能不是结构化 JSON，而是 Markdown 标题、粗体字段、下划线占位和复选框组成的文本表单。现有表单预览只识别 `json/form/schema` 代码块，导致这类文本仍按普通 Markdown 或代码样式展示。

同时长表格、代码块或表单文本会把 assistant 消息撑宽，造成当前对话栏目横向移动。

## 目标

- 识别 Markdown 文本表单，并渲染为只读表单预览。
- 继续支持结构化 JSON 表单预览。
- 限制首页 AI 对话消息和 Markdown 内容只能在当前栏目内展示，不撑开页面横向宽度。
- 普通代码块仍保留代码展示和内部横向滚动。

## 非目标

- 不执行 AI 动作建议。
- 不提交表单、不调用业务接口、不写入业务页面。
- 不要求后端改变输出格式。

## 影响范围

- `frontend/src/components/MarkdownRenderer.tsx`
- `frontend/src/styles/globals.css`

## 风险等级

低风险。仅调整前端展示和布局约束，不涉及 API、IPC、本机文件或高风险动作。

## 验收标准

- “标题 + 多个粗体字段”的 Markdown 文本表单可渲染为只读表单预览。
- 结构化 JSON 表单仍可渲染为只读表单预览。
- AI 回复中的长表单、表格或代码块不再撑开首页对话栏目。
- TypeScript 检查通过。
