# Optimize Project Page Reference

## 背景

用户提供参考截图，要求项目界面红框区域高保真还原：筛选、搜索、创建项目入口和项目列表卡片需要贴近参考图。

## 目标

- 优化项目页内容区视觉结构。
- 还原参考图中的筛选控件、搜索条、创建项目按钮和项目卡片列表。
- 保留项目编辑预留入口但不触发请求。
- 使用 5 条项目数据展示当前分页视图。

## 非目标

- 不接入真实后端接口。
- 不实现项目创建、编辑、删除和详情跳转。
- 不调整任务、数据、模型、设置等其他页面。

## 验收标准

- 项目页首屏包含两个筛选控件、搜索框和右侧“创建项目”按钮。
- 项目卡片按参考图纵向排列，展示图标、名称、状态、描述、路径、训练任务、部署任务、数据集和更新时间。
- 项目列表每页展示 5 个项目，并保留分页信息。
- 项目编辑只保留空按钮，不提供可见交互。
