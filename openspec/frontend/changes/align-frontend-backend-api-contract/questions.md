# Questions

## 已确认

1. 后端新版文档作为前端项目模块业务 API 的唯一准入标准；旧前端接口替换文档与其冲突时，全部以后端新版为准。
2. 项目编辑功能保留，但当前阶段暂不处理；前端只预留位置，不实现旧版 `PATCH /api/projects/{projectId}` 请求。
3. 当前阶段只针对项目模块，不处理安装初始化、模型下载、任务、数据、模型、环境、个人设置等其他模块问题。

## 项目模块待确认

1. 项目列表响应示例包含 `page`、`pageSize`、`total`、`hasMore`，未包含 `totalPages`；前端是否按 `totalPages` 可选处理？
2. 项目创建是否必须传 `confirmed=true`，且不需要 `Idempotency-Key`？当前以后端项目章节为准：只传 `confirmed=true`。
3. 项目工作区路径提交前是否必须调用路径校验接口，还是先只依赖目录选择/用户输入？
4. 项目编辑入口当前应展示为禁用、隐藏，还是显示“待接入”占位？
5. 项目详情页中的关联数据、任务、模型 Tab 是否本阶段隐藏，还是保留空态并提示后续接入？

## 暂不处理的问题

- 安装初始化接口 6.1-6.12 的前端归属。
- 模型下载接口 5.8-5.13 的 V1.0 范围和入口。
- 任务、数据、模型、环境、个人设置等模块的完整接入问题。
- 全局幂等键策略和多模块高风险动作策略。
