# Align Frontend To Backend API Contract

## 背景

后端接口文档已按新版原型冻结。当前阶段先处理项目模块，前端项目接口接入必须以该后端文档为准，避免继续沿用 mock 字段、旧接口编号或前端自定义临时契约。

## 目标

- 以后端新版接口文档作为项目模块业务 API 的唯一对接来源。
- 先完成项目模块 OpenSpec 提问优化，明确接口边界、预留能力和验收方式。
- 输出项目模块可执行接入阶段：契约层、服务层、页面替换、风险确认、验收。
- 项目编辑功能保留产品位置，但当前不实现接口调用。

## 非目标

- 本 Change 不立即实现全部接口接入。
- 不修改后端接口文档。
- 不新增未在后端文档中定义的业务接口。
- 不绕过后端执行本地路径、进程、环境、清理、下载或安装动作。
- 当前阶段不处理首页、任务、数据、模型、安装初始化、运行环境、个人设置和本地文件能力。
- 当前阶段不实现项目编辑接口，仅预留 UI / 类型 / 任务位置。

## 影响模块

- 项目：列表、规格、创建、详情、删除。
- 项目编辑：保留入口和后续任务占位，暂不接后端。
- AI 助手：项目相关真实动作只通过后端业务接口发起，不直接替代业务 API。

## 风险

- 当前项目页仍使用 mock 数据，替换时需要避免半真半假状态。
- 项目创建属于中风险操作，删除属于高风险操作，确认流程必须和后端 `confirmed` 校验一致。
- 项目编辑保留但暂不处理，若 UI 过早暴露可编辑提交能力，会形成无后端契约的假入口。

## 验收标准

- 项目模块待确认问题已收敛，并记录用户已确认决策。
- 项目模块接口映射到后端 2.1-2.5。
- 项目创建和删除的确认、错误映射、loading / empty / error 状态有统一规则。
- 项目编辑只保留位置，不进入当前实现任务。
