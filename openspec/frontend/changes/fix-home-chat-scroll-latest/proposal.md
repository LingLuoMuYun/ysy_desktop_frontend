# 修复首页 AI 对话进入时定位到最新回复

## 背景

首页 AI 对话已有历史会话和消息渲染能力，但用户从其他模块点击回首页，或切换到已有历史对话时，消息区会停留在历史顶部，用户需要手动滚动到底部才能看到最新回答。

## 目标

- 进入首页已存在对话时，消息区自动定位到最新一条消息底部。
- 首页消息新增或流式回复更新时，消息区保持跟随最新回复。
- 保持现有历史会话托管关系不变：`AppShell` 管理会话，`HomePage` 只处理消息展示和输入。

## 非目标

- 不接入真实后端 sessions 持久化。
- 不调整历史会话排序、标题生成或消息结构。
- 不修改右侧 AI 助手面板行为。

## 影响范围

- `frontend/src/pages/HomePage.tsx`

## 风险等级

低风险。该变更仅影响前端消息列表滚动定位，不执行本机动作，不修改 API、IPC、数据模型或高风险确认流程。

## 验收标准

- 首页存在历史消息时，进入首页后默认看到最新下方回复。
- 切换到已有历史对话后，消息区定位到最新消息。
- AI 回复流式更新时，消息区持续跟随最新内容。
- `pnpm run typecheck` 通过。
