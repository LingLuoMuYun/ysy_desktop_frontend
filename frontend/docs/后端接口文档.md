# 衍生桌面智算后端 API 接口说明文档
状态：新版原型功能模块对齐冻结版  
原型地址：`http://10.0.195.51:5175/`  
关联功能拆分：`docs/prd/feature-breakdown.md`  
说明：本文档按新版产品功能模块组织后端 API。每个接口均包含前置条件、说明和接口说明。本文档已对齐前端页面一级导航：首页、项目、任务、数据、模型、设置，以及任务页的创建训练任务、创建部署任务、任务概览、配置参数、日志与诊断、输出结果等操作入口。

## 0. 全局约定
### 0.1 基础约定
#### 1. 前置条件
+ **业务阶段**：桌面端 MVP 后端 API 契约设计。
+ **前置动作**：前端已切换为新版原型，一级导航为：首页、项目、任务、数据、模型、设置。
+ **基础路径**：所有业务接口默认使用 `/api` 前缀。

#### 2. 说明
+ 新版原型中，训练任务和部署服务统一归入“任务”模块。
+ 数据集和模型产物升级为独立资产模块。
+ 前端只提交结构化参数，不直接提交任意 shell 命令。
+ 本地路径、进程启动、环境修复、删除、清理等操作必须由后端做安全校验。
+ 高风险操作必须传入 `confirmed: true`。
+ 所有业务接口的实际响应必须使用统一响应包裹；下文示例中的“输出”若未显式写出 `success/data/error/requestId`，均表示 `data` 内部的业务载荷。
+ 创建、启动、停止、生成报告、执行依赖安装、创建模型下载任务等可能被重复点击或重试的接口支持 `Idempotency-Key` 请求头；也可在请求体中传入 `idempotencyKey`。同一幂等键在同一业务资源范围内必须返回同一业务结果或明确的幂等冲突错误。

#### 3. 接口说明
**统一成功输出**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `success` | `Boolean` | 固定为 `true` |
| `data` | `Object / Array / Null` | 业务数据 |
| `error` | `Null` | 成功时为空 |
| `requestId` | `String` | 请求追踪 ID |

**统一成功输出示例**

```json
{
  "success": true,
  "data": {},
  "error": null,
  "requestId": "req-20260702-0001"
}
```


**统一失败输出**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `success` | `Boolean` | 固定为 `false` |
| `data` | `Null` | 失败时为空 |
| `error.code` | `String` | 错误码 |
| `error.message` | `String` | 错误说明 |
| `error.detail` | `Object` | 错误详情 |
| `error.suggestion` | `String` | 下一步建议 |
| `requestId` | `String` | 请求追踪 ID |

**统一失败输出示例**

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "validation_error",
    "message": "参数校验失败",
    "detail": {},
    "suggestion": "请检查必填字段"
  },
  "requestId": "req-20260702-0002"
}
```


**通用错误码**

| 错误码 | 说明 |
| :--- | :--- |
| `unauthorized` | 未登录或 token 无效 |
| `validation_error` | 参数校验失败 |
| `not_found` | 资源不存在 |
| `path_not_found` | 本地路径不存在 |
| `path_not_allowed` | 路径超出允许范围 |
| `environment_not_ready` | 运行环境不可用 |
| `dataset_invalid` | 数据集格式或标注校验失败 |
| `model_invalid` | 模型文件或配置不可用 |
| `process_start_failed` | 本地任务进程启动失败 |
| `task_not_cancellable` | 当前任务不可停止 |
| `confirm_required` | 高风险操作缺少确认 |
| `idempotency_conflict` | 幂等键对应的历史请求与本次请求体不一致 |
| `future_not_supported` | 当前 MVP 暂不支持的高风险扩展能力 |
| `dependency_conflict` | 依赖冲突，需要用户确认处理 |
| `hardware_not_supported` | 本机硬件不满足能力或模型启动要求 |
| `setup_job_not_cancellable` | 当前安装任务不可取消 |
| `model_download_blocked` | 模型下载前资源校验不通过 |
| `model_download_failed` | 模型下载失败 |
| `model_download_not_cancellable` | 当前模型下载任务不可取消 |
| `internal_error` | 后端内部错误 |


**分页输入**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `keyword` | `String` | 搜索关键字 |
| `page` | `Integer` | 页码，默认 `1` |
| `pageSize` | `Integer` | 每页数量，默认 `20` |
| `sortBy` | `String` | 排序字段 |
| `sortOrder` | `String` | `asc` 或 `desc` |


**分页输出**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `items` | `Array` | 当前页数据 |
| `page` | `Integer` | 当前页码 |
| `pageSize` | `Integer` | 每页数量 |
| `total` | `Integer` | 总数量 |
| `totalPages` | `Integer` | 总页数 |

### 0.2 高风险操作
#### 1. 前置条件
+ **业务阶段**：删除、停止、修复、清理。
+ **前置动作**：前端已展示确认弹窗，用户明确确认。

#### 2. 说明
以下接口必须在请求体中传入 `confirmed: true`。前端需要展示确认摘要；后端必须自行校验确认字段，不能只依赖前端弹窗。

#### 3. 接口说明
**统一高风险确认请求**

```json
{
  "confirmed": true,
  "confirmText": "可选，用户二次确认文本",
  "confirmToken": "可选，由预览接口或后端生成",
  "riskSummary": "前端展示用风险摘要"
}
```

缺少 `confirmed: true` 时返回：

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "confirm_required",
    "message": "高风险操作缺少用户确认",
    "detail": { "requiredField": "confirmed" },
    "suggestion": "请展示确认弹窗，用户确认后重试"
  },
  "requestId": "req-confirm-001"
}
```

| 操作 | 接口 |
| :--- | :--- |
| 清理本地垃圾文件 | `POST /api/system/cleanup` |
| 删除项目 | `DELETE /api/projects/{projectId}` |
| 删除任务 | `DELETE /api/tasks/{taskId}` |
| 启动任务 | `POST /api/tasks/{taskId}/start` |
| 停止任务 | `POST /api/tasks/{taskId}/stop` |
| 删除数据集 | `DELETE /api/datasets/{datasetId}` |
| 删除模型 | `DELETE /api/models/{modelId}` |
| 修复环境 | `POST /api/environments/{envId}/repair` |
| 删除环境 | `DELETE /api/environments/{envId}` |
| 执行初始化依赖安装 | `POST /api/setup/install` |
| 安装具体任务能力依赖 | `POST /api/setup/capabilities/install` |
| 取消安装任务 | `POST /api/setup/jobs/{setupJobId}/cancel` |
| 创建模型下载任务 | `POST /api/models/downloads` |
| 取消模型下载任务 | `POST /api/models/downloads/{downloadId}/cancel` |

删除项目、任务、数据集、模型等记录时，MVP 默认只删除登记记录，不删除本地文件。环境删除存在单独策略：所有已登记环境都支持数据库软删除；当目标环境为 Conda 子环境时，用户二次确认后允许同时删除本地 Conda 环境和数据库登记记录。

### 0.3 通用检查结果结构
#### 1. 前置条件
+ **业务阶段**：数据检查、模型检查、环境检测、任务预检、推理服务兼容性检查。
+ **前置动作**：用户触发检查，或导入/创建后自动检查。

#### 2. 说明
数据、模型、环境、任务预检和部署兼容性统一使用 `CheckResult`，便于前端复用检查结果组件。

#### 3. 接口说明
**CheckResult**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `status` | `String` | `pass`、`warning`、`failed`、`checking` |
| `summary` | `String` | 检查摘要 |
| `items` | `Array` | 检查项列表 |
| `items[].name` | `String` | 检查项名称 |
| `items[].status` | `String` | `pass`、`warning`、`failed`、`skipped` |
| `items[].severity` | `String` | `info`、`warning`、`error` |
| `items[].code` | `String` | 结构化问题码 |
| `items[].message` | `String` | 问题说明 |
| `items[].suggestion` | `String` | 建议动作 |
| `items[].blocking` | `Boolean` | 是否阻断创建或启动 |
| `items[].value` | `String / Number / Object / Null` | 检查值 |
| `items[].evidence` | `Array` | 证据列表，如文件路径、版本号、日志片段或检测值 |
| `checkedAt` | `String` | 最近检查时间 |
| `expiresAt` | `String` | 结果过期时间，可选 |
| `source` | `String` | `dataset`、`model`、`environment`、`task_preflight`、`deployment_compatibility` |
| `warnings` | `Array` | 非阻断警告 |

**示例**

```json
{
  "status": "warning",
  "summary": "存在 1 个非阻断警告，可继续创建但启动前需再次确认",
  "items": [
    {
      "name": "数据集检查",
      "status": "warning",
      "severity": "warning",
      "code": "LABEL_FILE_MISSING",
      "message": "发现 3 个标签文件缺失",
      "suggestion": "补齐标签或在训练前排除对应样本",
      "blocking": false,
      "value": 3,
      "evidence": ["labels/train/0001.txt", "labels/train/0042.txt"]
    }
  ],
  "checkedAt": "2026-07-02T15:30:00+08:00",
  "expiresAt": "2026-07-02T16:30:00+08:00",
  "source": "task_preflight",
  "warnings": ["3 个标签文件缺失"]
}
```

### 3.4 创建任务
#### 1. 前置条件
+ **业务阶段**：创建训练任务或部署任务。
+ **前置动作**：预检通过或用户选择允许带警告创建。

#### 2. 说明
统一创建训练任务和部署任务。`kind=training` 进入训练 Adapter，`kind=deployment` 进入部署 Adapter。创建任务时必须保存不可变的执行快照，后续项目、环境或模型记录变化不得修改已创建任务的快照。

#### 3. 接口说明
**接口地址**

`POST /api/tasks`

**幂等**

支持 `Idempotency-Key` 请求头或请求体 `idempotencyKey`，用于防止前端重复点击“创建训练任务 / 创建部署任务”导致重复任务。

**TaskExecutionSnapshot 字段**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `source` | `String` | `project_default`、`historical_task`、`local_script`、`custom_command` |
| `codePath` | `String` | 代码目录，可选 |
| `workdir` | `String` | 命令执行工作目录，必填 |
| `commandTemplate` | `String` | 命令模板，不允许前端提交任意未校验 shell 片段 |
| `resolvedCommand` | `String` | 后端解析模板后的最终命令，必须落库用于复现 |
| `args` | `Array` | 参数数组 |
| `envVars` | `Object` | 注入环境变量 |
| `outputPath` | `String` | 输出目录 |
| `interpreterPath` | `String` | 任务绑定环境的 Python 解释器路径 |
| `environmentId` | `String` | 运行环境 ID |
| `dependencyFile` | `String` | 依赖文件路径，可选 |
| `configFile` | `String` | 配置文件路径，可选 |
| `timeoutSeconds` | `Integer` | 超时时间，`0` 表示不限制 |
| `logRef` | `String` | 日志存储引用 |
| `artifactScanRules` | `Array` | 产物扫描规则 |

`commandTemplate` 只允许以下模板变量：`taskId`、`projectId`、`datasetPath`、`modelPath`、`outputPath`、`envName`。后端必须在预检和启动前解析并保存 `resolvedCommand`。

### 7.2 创建环境

#### 1. 前置条件

- **业务阶段**：创建系统环境或自定义环境。
- **前置动作**：用户在“创建系统环境”或“自定义环境”表单中确认创建。

#### 2. 说明

创建运行环境统一使用 `POST /api/environments`，通过请求体中的 `mode` 区分创建方式。

- `mode=system`：创建系统环境。当前版本支持“大模型推理 / LLM”，后端自动生成环境名称和 Conda 环境名称，并在 Conda 默认 `envs` 目录下创建环境，同时安装系统预置的 llama.cpp 框架包。
- `mode=custom`：创建自定义环境。用户需要提供环境名称、用途、Python 版本、环境创建方式、保存路径、依赖文件或依赖包列表。
- 创建环境属于高风险、长耗时操作，必须传入 `confirmed=true` 和请求头 `Idempotency-Key`。
- 接口只负责登记环境记录并创建后台任务，HTTP 请求不会等待环境创建和依赖安装全部完成。
- 新建环境初始状态为 `creating`。后台任务执行完成后，如果 `autoCheck=false`，环境状态更新为 `unchecked`；如果 `autoCheck=true`，后续进入检测流程并更新为 `checking`、`available` 或 `unavailable`。

#### 3. 接口说明

**接口地址**

`POST /api/environments`

**请求头**

| 参数名 | 类型 | 是否必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `Idempotency-Key` | `String` | 是 | 幂等键。用于避免用户重复点击创建按钮导致重复创建环境。同一幂等键和相同请求体应返回同一个业务结果。 |

### 7.3 导入本地环境
#### 1. 前置条件
+ **业务阶段**：导入本地环境。
+ **前置动作**：用户选择已有 Conda / venv。

#### 2. 说明
登记本地环境，不复制、不创建、不修改用户已有 Python 环境。前端“导入本地环境”表单中的环境名称、环境类型、环境用途、本地环境路径、Python 解释器路径、Conda 环境名和自动检测开关必须全部通过结构化参数提交给后端。

前端“环境类型”字段表示环境管理器，对应接口字段 `environmentManager`，可选值为 `conda`、`venv`、`python`。数据库字段 `environment_type` 不由前端直接提交，由后端根据 `purpose` 推导，例如：`训练 -> training`、`推理 -> inference`、`LLM 微调 -> llm_finetune`、`LLM 推理 -> llm_inference`，无法识别时为 `general`。

后端必须校验本地环境路径和 Python 解释器路径的一致性：`pythonPath` 必须位于 `environmentPath` 目录内，或符合后端认可的 Conda / venv 子路径结构。`environmentManager` 为 `conda` 时，`condaEnvName` 可为空；为空时后端可从路径或 Conda 探测结果推导。导入接口属于低风险登记操作，不安装依赖、不修复环境、不删除文件。

#### 3. 接口说明
**接口地址**

`POST /api/environments/import`

**输入**

```json
{
  "name": "PyTorch 通用训练环境",
  "purpose": "训练",
  "environmentPath": "D:/envs/pytorch-general",
  "pythonPath": "D:/envs/pytorch-general/python.exe",
  "environmentManager": "conda",
  "condaEnvName": "pytorch-general",
  "autoCheck": true
}
```

| 参数名 | 类型 | 必填 | 对应前端字段 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `name` | `String` | 是 | 环境名称 | 环境显示名称，前端可自动生成并允许用户修改 |
| `purpose` | `String` | 是 | 环境用途 | 用户可读用途说明，对应数据库 `usage_scope`，如 `训练`、`推理`、`LLM 微调` |
| `environmentPath` | `String` | 是 | 本地环境路径 | 本地环境目录，对应数据库 `environment_path` |
| `pythonPath` | `String` | 是 | Python 解释器路径 | Python 可执行文件路径；后端必须校验存在、是文件且可执行 |
| `environmentManager` | `String` | 是 | 环境类型 | 环境管理器，对应数据库 `environment_manager`；可选值：`conda`、`venv`、`python` |
| `condaEnvName` | `String` | 否 | Conda 环境名 | Conda 环境名称；`environmentManager=conda` 时建议传入，其他类型为空 |
| `autoCheck` | `Boolean` | 是 | 创建 / 导入后自动检测 | 是否在导入后执行轻量检测；不直接落库，影响 `environment_status`、`check_result_json`、`checked_at` |

**后端推导字段**

| 参数名 | 类型 | 来源 | 说明 |
| :--- | :--- | :--- | :--- |
| `environmentType` | `String` | 后端根据 `purpose` 推导 | 运行环境类型，对应数据库 `environment_type`；可选值：`training`、`inference`、`llm_finetune`、`llm_inference`、`general` |

**校验规则**

| 规则 | 失败错误码 | 说明 |
| :--- | :--- | :--- |
| `environmentPath` 必须存在且为目录 | `path_not_found` / `validation_error` | 路径不存在返回 `path_not_found`，不是目录返回 `validation_error` |
| `pythonPath` 必须存在且为文件 | `path_not_found` / `validation_error` | 路径不存在返回 `path_not_found`，不是文件返回 `validation_error` |

| 同一 `pythonPath` 或 `environmentPath` 不可重复导入 | `validation_error` | 路径归一化后先比较 `pythonPath`，再比较 `environmentPath` |

**状态规则**

### 7.6 删除环境
#### 1. 前置条件
+ **业务阶段**：环境删除。
+ **前置动作**：用户确认删除。

#### 2. 说明
删除环境前必须先检查环境来源和环境管理器：

+ `deleteLocalFiles=false`：只软删除数据库登记记录，写入 `deleted_at`，不删除任何本地文件。
+ `deleteLocalFiles=true` 且 `deleteLocalFilesConfirmed=false`：返回 `local_delete_confirmation_required`，前端必须弹出二次确认框提示用户会删除本地 Conda 环境。
+ `deleteLocalFiles=true` 且 `deleteLocalFilesConfirmed=true`，并且目标为 Conda 子环境：先执行受控 Conda 环境删除，再软删除数据库登记记录。
+ `deleteLocalFiles=true` 且目标不是 Conda 子环境：返回 `local_delete_not_allowed`，不得删除本地环境文件，也不得写入 `deleted_at`。
+ 本地 Conda 环境删除失败时，返回 `local_environment_delete_failed` 或 `conda_executable_not_found`，不得写入 `deleted_at`。

#### 3. 接口说明
**接口地址**

`DELETE /api/environments/{envId}`

**输入**

```json
{
  "confirmed": true,
  "deleteLocalFiles": false
}
```

**输出**

```json
{
  "deleted": true,
  "localFilesDeleted": false
}
```

**软删除并删除本地 Conda 环境输入**

```json
{
  "confirmed": true,
  "deleteLocalFiles": true,
```

#### 3. 接口说明
| 规则 | 说明 |
| :--- | :--- |
| bbox 宽度校验 | `x_max <= x_min` 的框必须过滤 |

| COCO 输入 | 使用 `[x, y, width, height]` |
| YOLO 输入 | 使用归一化 `[cx, cy, w, h]` 并转换 |

| 空标注处理 | 某张图所有框被过滤时，按后端策略跳过或保留空标注 |


### 10.3 任务状态
#### 1. 前置条件
+ **业务阶段**：任务生命周期管理。
+ **前置动作**：任务已创建。

#### 2. 说明
训练任务和部署任务统一维护 `status` 主状态和 `phase` 执行阶段。`status` 是任务生命周期唯一主状态；`phase` 只表示当前执行步骤，不替代主状态。

任务主状态不包含 `unavailable`。数据集、模型、运行环境的不可用属于资源状态；任务侧通过预检 `blockingReasons`、`errorCode` 和 `errorMessage` 表达阻断原因。

#### 3. 接口说明
**任务主状态**

| `apiStatus` | 中文展示 | PRD 状态 | 适用 kind | 是否终态 | 允许操作 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `draft` | 等待配置 | 等待配置 | training / deployment | 否 | 编辑、删除 |
| `pending` | 待启动 | 待启动 | training / deployment | 否 | 启动、编辑、复制、删除 |
| `checking` | 检查中 | 检查中 | training / deployment | 否 | 查看、停止检查 |
| `queued` | 排队中 | 排队中 | training / deployment | 否 | 查看、停止 |
| `running` | 运行中 | 运行中 | training / deployment | 否 | 停止、复制 |
| `stopping` | 停止中 | 停止中 | training / deployment | 否 | 查看 |
| `stopped` | 已停止 | 已停止 | training / deployment | 是 | 复制、删除 |
| `completed` | 已完成 | 已完成 | training | 是 | 复制、生成报告、登记模型、删除 |
| `start_failed` | 启动失败 | 启动失败 | training / deployment | 是 | 编辑、复制、删除、AI 诊断 |
| `failed` | 失败 | 失败 | training / deployment | 是 | 复制、生成失败诊断报告、删除、AI 诊断 |
| `error` | 异常 | 异常 | training / deployment | 是 | 复制、删除、AI 诊断 |

**任务执行阶段 `phase`**

| `phase` | 前端展示 | 适用 kind | 说明 |
| :--- | :--- | :--- | :--- |
| `environment_check` | 环境检查中 | training / deployment | 检查 Python、依赖、CUDA |
| `dataset_check` | 数据检查中 | training | 检查训练数据 |
| `model_check` | 模型检查中 | deployment | 检查模型文件、tokenizer、adapter |
| `resource_check` | 资源检查中 | training / deployment | 检查 GPU、内存、磁盘、端口 |
| `queue` | 排队中 | training / deployment | 等待执行资源 |
| `startup` | 启动中 | training / deployment | 创建训练进程或服务进程 |
| `model_loading` | 模型加载中 | deployment | 推理服务加载模型 |
| `service_start` | 服务启动中 | deployment | 推理服务启动 HTTP endpoint |
| `health_check` | 健康检查中 | deployment | 检查服务 endpoint |
| `training` | 训练中 | training | 训练主过程 |
| `validation` | 验证评估中 | training | 验证集评估 |
| `artifact_scan` | 导出产物中 | training | 扫描 checkpoint、报告候选和模型文件 |

报告生成不属于任务 `phase`。报告状态独立维护在 `Report.status` 中。

**训练任务状态规则**

1. 创建后为 `pending`；若表单未完成可为 `draft`。
2. 启动后依次进入 `checking`、`queued`、`running`。
3. 训练正常退出并完成产物扫描后进入 `completed`。
4. 启动前失败进入 `start_failed`。
5. 训练进程非 0 退出进入 `failed`。
6. 未知执行器异常、状态同步异常或进程丢失进入 `error`。
7. 用户停止后进入 `stopping`，进程结束后进入 `stopped`，`stoppedReason=USER_STOPPED`。

**部署任务状态规则**

1. 创建后为 `pending`；若表单未完成可为 `draft`。
2. 启动后依次进入 `checking`、`queued`、`running`。
3. 健康检查通过后保持 `running`，不会进入 `completed`。
4. 端口冲突、模型缺文件、基础模型不匹配、依赖缺失等启动前问题进入 `start_failed`。
5. 服务进程启动后崩溃进入 `error`。
6. 用户停止后进入 `stopping`，进程结束后进入 `stopped`。

**错误字段**

| 参数名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `errorCode` | `String` | 结构化错误码，例如 `COMMAND_NOT_FOUND`、`CUDA_OOM`、`PORT_IN_USE` |
| `errorMessage` | `String` | 错误摘要 |
| `exitCode` | `Integer` | 进程退出码 |
| `stoppedReason` | `String` | `USER_STOPPED`、`PROCESS_EXITED`、`SYSTEM_SHUTDOWN` |
