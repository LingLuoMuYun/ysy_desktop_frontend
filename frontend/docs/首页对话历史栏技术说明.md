# 首页对话历史栏技术说明

## 功能范围

首页顶部栏的“对话历史”按钮用于打开历史栏。历史栏显示在左侧导航右侧、首页主对话区左侧，用户可以查看已产生消息的对话并点击恢复；“新对话”用于创建空白当前会话。

当前实现是前端内存态，用于完成页面交互闭环。真实 AI 会话持久化接入后，应替换数据来源，不改变历史栏的展示边界。

## 相关文件

- `frontend/src/layouts/AppShell.tsx`：托管首页会话列表、当前会话、新对话、历史栏开关和会话切换。
- `frontend/src/layouts/WindowTitleBar.tsx`：提供“对话历史”和“新对话”按钮事件入口。
- `frontend/src/layouts/ConversationHistoryPanel.tsx`：展示历史列表、空态和当前选中态。
- `frontend/src/layouts/conversationTypes.ts`：定义历史会话展示模型。
- `frontend/src/pages/HomePage.tsx`：渲染当前会话消息，发送消息后把消息列表回传给 AppShell。
- `frontend/src/styles/globals.css`：定义历史栏布局、选中态和响应式 Grid 列。
- `frontend/src/styles/tokens.css`：定义 `--history-width` 默认宽度变量。

## 状态流

```text
WindowTitleBar
  对话历史 -> AppShell.conversationHistoryOpen
  新对话   -> AppShell.createEmptyConversation()

HomePage
  发送消息 -> onMessagesChange(messages, title)

AppShell
  更新当前会话
  最近更新置顶
  空白会话不传给历史列表
  将 activeConversation.messages 注入 HomePage

ConversationHistoryPanel
  点击条目 -> AppShell.activeHomeConversationId
```

## 布局规则

AppShell 使用 CSS Grid。首页历史栏是独立列，不是浮层：

```text
左侧导航 | 对话历史栏 | 主工作区 | 右侧上下文栏
```

历史栏关闭时 `--history-width: 0px`；打开时 `.app-window--history-open` 将宽度设为 `236px`。响应式规则中也必须保留历史列，否则历史栏会覆盖主工作区。

维护样式时重点检查：

- `.app-window`
- `.app-window--assistant-closed`
- `.app-window--menu-collapsed`
- `@media (max-width: 1280px)`
- `@media (max-width: 1120px)`
- `.conversation-history`
- `.app-surface`
- `.local-resource-panel`
- `.assistant-panel`

## 交互规则

- 点击“对话历史”：切换历史栏显隐。
- 点击“新对话”：创建空白会话，首页回到初始输入态。
- 空白新对话不展示在历史列表。
- 首次发送消息后，会话进入历史列表并置顶。
- 点击历史条目：恢复该会话消息，顶部栏标题同步为会话标题。
- 离开首页时，资源概览、历史栏和首页标题重置；会话内存态保留在当前前端运行周期内。

## 后续接入真实接口

根据 `frontend/docs/智能体接口.md`，历史会话接口包括：

- `GET /api/sessions`
- `GET /api/sessions/{session_key}`
- `DELETE /api/sessions/{session_key}`

建议接入方式：

1. 在 home feature 或 service 层新增 session 适配器。
2. 将后端 `session.key` 映射为 `ConversationSummary.id` 或新增 `sessionKey` 字段。
3. 打开历史栏时加载 sessions 列表。
4. 点击历史条目时加载详情并映射为 `ChatMessage[]`。
5. 保持 `ConversationHistoryPanel` 不感知接口 DTO。

## 验证建议

修改该功能后至少验证：

- `npm run build`
- 首页发送消息后历史栏出现条目。
- 点击“新对话”后首页回到空白输入态。
- 点击历史条目后恢复用户消息和助手消息。
- 1280px 左右宽度下历史栏不覆盖主工作区。
- 右侧资源概览打开时，历史栏、主工作区、右侧栏三者不重叠。
