# 首页对话历史栏技术说明

## 功能范围

首页顶部栏的”对话历史”按钮用于打开历史栏。历史栏显示在左侧导航右侧、首页主对话区左侧，用户可以查看已产生消息的对话并点击恢复；”新对话”用于创建空白当前会话。

**V1.1 新增**：
- 历史栏右侧支持**拖拽调整宽度**（220px - 500px，默认 236px）
- 历史栏条目支持**双击/右键重命名**
- 顶部栏当前对话标题支持**双击/右键重命名**
- 标题字体缩小（13px / 不加粗），更紧凑

当前实现是前端内存态，用于完成页面交互闭环。真实 AI 会话持久化接入后，应替换数据来源，不改变历史栏的展示边界。

## 相关文件

- `frontend/src/layouts/AppShell.tsx`：托管首页会话列表、当前会话、新对话、历史栏开关、历史栏宽度状态（`historyPanelWidth`）和拖拽事件处理。
- `frontend/src/layouts/WindowTitleBar.tsx`：提供”对话历史”和”新对话”按钮事件入口；当前对话标题支持双击/右键进入内联重命名。
- `frontend/src/layouts/ConversationHistoryPanel.tsx`：展示历史列表、空态、当前选中态、内联重命名和右键菜单（重命名/删除）。
- `frontend/src/layouts/conversationTypes.ts`：定义历史会话展示模型。
- `frontend/src/pages/HomePage.tsx`：渲染当前会话消息，发送消息后把消息列表回传给 AppShell。
- `frontend/src/styles/globals.css`：定义历史栏布局、选中态、拖拽手柄（`.history-panel-resizer`）和响应式 Grid 列。
- `frontend/src/styles/tokens.css`：定义 `--history-width` 默认宽度变量。
- `frontend/src/styles/windowControls.css`：Electron 窗口拖拽区域配置，标题栏重命名元素需设置 `no-drag`。

## 状态流

```text
WindowTitleBar
  对话历史 -> AppShell.conversationHistoryOpen
  新对话   -> AppShell.createEmptyConversation()

HomePage
  发送消息 -> onMessagesChange(messages, title)

AppShell
  更新当前会话
  最近更新置顶
  空白会话不传给历史列表
  将 activeConversation.messages 注入 HomePage

ConversationHistoryPanel
  点击条目 -> AppShell.activeHomeConversationId
```

## 布局规则

AppShell 使用 CSS Grid。首页历史栏是独立列，不是浮层：

```text
左侧导航 | 对话历史栏 | 主工作区 | 右侧上下文栏
```

历史栏宽度由 CSS 变量 `--history-width` 控制：
- 关闭时：`0px`（由 `tokens.css` 默认值和 AppShell inline style 共同控制）
- 打开时：默认 `236px`，用户可向右拖拽至 `220px - 500px`
- 宽度由 AppShell 的 `historyPanelWidth` 状态管理，通过 inline style 注入 `--history-width`

拖拽手柄 `.history-panel-resizer` 定位在历史栏右边缘，使用 `col-resize` 光标，hover 时显示蓝色竖线。

响应式规则中也必须保留历史列，否则历史栏会覆盖主工作区。

维护样式时重点检查：

- `.app-window`
- `.app-window--history-open`
- `.app-window--assistant-closed`
- `.app-window--menu-collapsed`
- `@media (max-width: 1280px)`
- `@media (max-width: 1120px)`
- `.conversation-history`
- `.history-panel-resizer`
- `.app-surface`
- `.local-resource-panel`
- `.assistant-panel`

## 交互规则

- 点击”对话历史”：切换历史栏显隐。
- 点击”新对话”：创建空白会话，首页回到初始输入态。
- 空白新对话不展示在历史列表。
- 首次发送消息后，会话进入历史列表并置顶。
- 点击历史条目：恢复该会话消息，顶部栏标题同步为会话标题。
- **拖拽历史栏右边缘**：调整历史栏宽度（220px - 500px），释放鼠标后宽度保持。
- **双击顶部标题 / 右键标题**：进入内联重命名模式，Enter 确认、Escape 取消。
- **双击历史条目 / 右键条目**：进入内联重命名或选择”重命名”/”删除”。
- 离开首页时，资源概览、历史栏和首页标题重置；会话内存态和面板宽度保留在当前前端运行周期内。

## 后续接入真实接口

根据 `frontend/docs/智能体接口.md`，历史会话接口包括：

- `GET /api/sessions`
- `GET /api/sessions/{session_key}`
- `DELETE /api/sessions/{session_key}`

建议接入方式：

1. 在 home feature 或 service 层新增 session 适配器。
2. 将后端 `session.key` 映射为 `ConversationSummary.id` 或新增 `sessionKey` 字段。
3. 打开历史栏时加载 sessions 列表。
4. 点击历史条目时加载详情并映射为 `ChatMessage[]`。
5. 保持 `ConversationHistoryPanel` 不感知接口 DTO。
