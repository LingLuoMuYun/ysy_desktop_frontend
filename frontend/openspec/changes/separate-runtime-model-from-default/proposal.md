# 区分使用模型和默认模型

## 背景

首页 AI 对话中的模型选择表示本次运行时“使用模型”，不应修改设置页中的默认模型。默认模型只有在设置页点击“设为默认”后变化；默认模型变化时，首页当前使用模型需要自动跟随新的默认模型。

## 目标

- 首页和右侧 AI 助手切换模型时，只更新前端当前使用模型。
- 不从首页模型选择入口调用设置默认模型接口。
- 对话发送时通过 `metadata.modelConfigId` 指定当前使用模型。
- 刷新模型列表时，只有检测到默认模型 ID 变化，才自动把当前使用模型切换为新的默认模型。
- 设置页“设为默认”成功后，当前使用模型同步到新的默认模型。

## 非目标

- 不新增后端接口。
- 不改变设置页模型默认值保存契约。
- 不持久化首页临时使用模型。

## 影响范围

- `frontend/src/layouts/AssistantPanelContext.tsx`
- `frontend/src/pages/HomePage.tsx`
- `frontend/src/services/chatApi.ts`

## 风险等级

低风险。该变更只调整前端模型状态选择逻辑，不执行本机高风险动作。

## 验收标准

- 首页选择模型只切换当前使用模型，不调用默认模型接口，也不调用会改写默认状态的运行时切换接口。
- 设置页改变默认模型后，首页使用模型自动变为新的默认模型。
- 首页和右侧 AI 助手发送消息时携带当前使用模型 ID。
- 普通模型列表刷新不会覆盖用户手动选择的非默认使用模型。
- `pnpm run typecheck` 通过。
